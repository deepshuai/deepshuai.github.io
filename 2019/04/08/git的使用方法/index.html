认识版本控制系统

版本控制系统分为三类：1、本地版本控制系统、2、集中化的版本控制系统（ Centralized Version Control Systems，简称 CVCS ）、3、分布式版本控制系统（ Distributed Version Control System，简称 DVCS ）。
1、本地版本控制系统： 如rcs，本地保存并管理补丁(patch)，文件补丁是一种特定的文件格式，记录文件修订前后内容的变化。适合个人本地版本管理，使用场景比较有限。
2、集中化的版本控制系统： 如CVS，Subversion 以及 Perforce等，集中式顾名思义有一个单一的集中管理的服务器，保存所有文件的修订版本，而协同工作的人们通过客户端连接到这台服务器上，取出最新的文件进行修改并提交更新。好处：更容易控制权限，任务分工更明确；坏处：单点故障问题；单队列的任务提交形式效率很低。
3、分布式版本控制系统： 如Git，Mercurial，Bazaar，Darcs等，不同于集中式版本控制库仅有中央服务器有一份完整的备份，客户端会提取代码仓库的完整备份。

Git文件三种状态

这是一个很重要的概念，后续的操作主要就是对文件的状态做修改或者在文件状态的基础上执行操作，是理解Git背后逻辑的关键。三种状态分别是：已修改(modified)，已暂存(staged)，已提交(committed)，三种状态的文件流转对应三个工作区域：Git的工作目录，暂存区域，本地仓库。
